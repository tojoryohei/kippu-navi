# JR最安分割乗車券 探索システム

JRの複雑な運賃体系をアルゴリズムで解析し，最も安価な「分割乗車券」の組み合わせを瞬時に算出するWebアプリケーションです．大学の卒業研究の一環として，計算量が多くなりがちな経路探索の最適化と，実用的なUXの両立をテーマに開発しました．



---

## 概要

JRでは，特定の駅で切符を分割して購入すると，通しの正規運賃よりも安くなるケースが存在します．しかし，無数にある駅の組み合わせから最適な分割点を見つけ出すのは人間には困難です．
本システムは，出発駅と到着駅を入力するだけで，計算幾何学と動的計画法を活用し，数ミリ秒〜数秒のレスポンスで「大域的最適解（最も安い分割パターン）」を提示します．

---

## システムアーキテクチャとアルゴリズム

実用的な処理速度と保守性を担保するため，以下のアルゴリズムとアーキテクチャを採用しています．

* **経路候補の列挙 (Yen's Algorithm)**
  単一の最短経路だけでなく，K-最短経路アルゴリズムを用いて，運賃計算の仕様上「遠回りした方が安くなる」可能性のあるルート候補を複数列挙します．理論上の最安値（限界距離）を用いた枝刈りを行うことで，計算の爆発を防いでいます．
* **最適分割の算出 (1次元動的計画法 - DP)**
  列挙された各経路に対して1次元DPを適用し，部分構造最適性に基づき最安の分割ポイントを計算します．同額となる別の分割パターンもロスなく保持するロジックを組んでいます．
* **Firestoreを活用した多層キャッシュ**
  計算コストの重い処理であるため，一度計算した同一区間の結果をFirebase Firestoreに非同期で保存しています．2回目以降の同一リクエストに対しては計算プロセスをスキップし，数十ミリ秒での応答を実現しています．

## 工夫した点

* **浮動小数点数エラーの完全排除**
  運賃計算の要となる「営業キロ」の計算において，JavaScript特有の少数演算の誤差（IEEE 754）を防ぐため，値をすべて10倍して整数（`number`型）として管理・計算する堅牢な設計にしています．
* **ノンブロッキングなキャッシュ書き込み**
  Firestoreへのキャッシュ保存時，クライアントへのレスポンス完了を待たずに（`await`せずに）バックグラウンドで書き込みを行うことで，初回実行時のUXを損なわないよう配慮しています．
* **インクリメンタルサーチの導入**
  フロントエンドのUIには `react-select` 等を活用し，駅名の逐次検索（タイピングごとの候補表示）を実装．ユーザーの入力ストレスを軽減しています．

## 使い方

以下のURLから実際にシステムをご利用いただけます．

**[https://kippu-navi.com/split/](https://kippu-navi.com/split/)**

1. 出発駅と到着駅をフォームに入力します．
2. 計算ボタンを押すと，通し運賃と最適な分割運賃が比較表示されます．
3. （※同じ駅間を再度検索すると，キャッシュ機構により一瞬で結果が返ることを確認できます）
